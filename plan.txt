# HumanEnv Quick-Add Feature Plan

## Overview

Add anonymous, time-limited (24h) quick-add URLs that allow humans to create envs, API keys, and approve fingerprints via simple HTML pages. Used by agents when they need credentials that don't exist.

## Context

When an agent tries to get a secret that doesn't exist, instead of failing silently, we return a URL the human can open to create the missing credential. This unblocks agents without requiring them to explain what to do.

## Quick-Add URLs

### Server Mode
When agent requests missing key:
```
Error: { code: "ENV_KEY_MISSING", quickAddUrl: "http://server:3056/api/quick-add/env/:token" }
```

### Local Mode
When agent requests missing key:
```
Error: { code: "ENV_KEY_MISSING", quickAddCommand: "humanenv local set KEY <value>" }
```

## Token Design

- **Expiry**: 24 hours
- **Storage**: Hashed token in DB (plain token in URL)
- **Actions**: env, apikey, whitelist
- **One-time use**: Marked as used after successful action
